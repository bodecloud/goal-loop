Verifier-backed /goal loops for Cursor.

Goal Loop is intentionally small: describe an objective, define a shell verifier, and let the stop hook keep the agent moving until the verifier passes.

Try it View source
  1. 1Start/goal "Fix build"
  2. 2Verifynpm run build
  3. 3Continue or stopfollowup_message | {}

What it does

The verifier is the completion authority.

The agent can write code, but it cannot declare the goal done. Goal Loop runs your verifier after each completed agent turn. Failure becomes the next instruction. Success ends the loop.

Quickstart

One command and one verifier.

/goal Fix the app build --verify "npm run build"

Enable Cursor Agent Auto-run for unattended continuation. Without Auto-run, Goal Loop still verifies, but you may need to continue manually.

Command API

The public surface is four commands.

/goal <objective>

Creates an active goal and starts verifier-backed iteration.

/plan [objective]

Drafts an objective and verifier before activating the loop.

/goal-status

Shows status, iteration, verifier commands, and the last log path.

/goal-abort

Marks the current goal aborted so the stop hook returns {}.

Goal contract

.cursor/goal/active.json

Runtime state is project-local. Commit defaults.json if a repo has a shared verifier; ignore active.json and runs/. 

{
  "version": 1,
  "status": "active",
  "objective": "Fix the app build",
  "verify": {
    "commands": ["npm run build"],
    "cwd": ".",
    "timeout_ms": 600000
  },
  "limits": {
    "max_iterations": 20,
    "max_wall_ms": 7200000
  },
  "iteration": 0,
  "last_verify": null
}

Schema notes

Only the fields you need to operate the loop.

Field Purpose
objective The target the agent works toward.
verify.commands Shell commands run sequentially after each completed turn.
limits Iteration and wall-clock guardrails against runaway loops.
last_verify.log_path The log the agent reads after a failed verifier run.

Stop hook contract

Two outputs.

The hook returns {} when there is no more work for the loop. It returns followup_message when the verifier failed and Cursor should continue.

Failure output

{
  "followup_message": "Goal Loop verification failed..."
}

Success output

{}

Install

Use the plugin locally or from the marketplace when published.

Cursor Agent CLI

cursor-agent --plugin-dir "$PWD" --workspace /path/to/project

Cursor Marketplace

/add-plugin goal-loop

Available after marketplace approval.

Other agents

The pattern is portable.

Any agent runner can implement Goal Loop: persist the contract, run the verifier after a turn, feed failure logs back as the next instruction, and stop on success.
